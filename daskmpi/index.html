    Dask-mpi on ThetaGPU

    How to run Dask-MPI on ThetaGPU at Argonne Leadership Computing Facility.

    • Install
    • Submit a batch job
    • Run a script in an interactive session
    • Start a JupyterLab interactive session

    Install

    1. ssh into one of Theta's login nodes

      ssh username@theta.alcf.anl.gov

    2. ssh into one of ThetaGPU's login nodes

      ssh thetagpusn1

    3. Load a pre-defined conda environment (either one works fine)

      with pytorch 

      source /lus/theta-fs0/software/thetagpu/conda/pt_master/2020-11-25/mconda3/setup.sh

      with tensorflow 

      source /lus/theta-fs0/software/thetagpu/conda/tf_master/2020-11-11/mconda3/setup.sh

    4. Create a new conda environment

      conda create -n envname

    5. Activate the environment

      conda activate envname

    6. Install required packages

      conda install -c rapidsai -c nvidia -c numba -c conda-forge \
cudf cupy python=3.7 ipython=5.8 cudatoolkit mpi4py dask dask-mpi \
bokeh jupyter Jupyterlab ipykernel

    7. Download the files start_daskmpi.py, cupy_test.py and jupyterlab_cupy_test.ipynb from this repository into a directory, e.g. dirname.

    8. Optional: create the Jupyter kernel for interactive JupyterLab sessions

      env=$(basename `echo $CONDA_PREFIX`)
python -m ipykernel install --user --name "$env" --display-name "Python [conda env:"$env"]"

    Local storage

    If your dataset is larger than the combined memory of all compute nodes, Dask will spill excess data to disk. If you do not have write permission to local storage on the compute nodes, spilling to disk will be disabled by default, as explained here.

    Submit a batch job

    This will run cupy_test.py using 8 GPUs on one node.

    1. ssh into one of Theta's login nodes

      ssh username@theta.alcf.anl.gov

    2. ssh into one of ThetaGPU's login nodes

      ssh thetagpusn1

    3. Submit a batch job

      • on a single node

        qsub -n 1 -q full-node -A datascience -t 00:30:00 daskmpi_job_gpu.sh

        where the script daskmpi_job_gpu.sh is

        #!/bin/bash
source /lus/theta-fs0/software/thetagpu/conda/pt_master/2020-11-25/mconda3/setup.sh
conda activate envname
cd dirname
export PYTHONPATH='/full/path/to/dirname'
mpirun -np 9 python start_daskmpi.py cupy_test.py

        The number of processors (-np) should be (at least) the number of GPUs per node plus one.

      • on multiple nodes

        qsub -n 2 -q full-node -A datascience -t 00:30:00 daskmpi_job_gpu.sh

        where the script daskmpi_job_gpu.sh is

        #!/bin/bash
source /lus/theta-fs0/software/thetagpu/conda/pt_master/2020-11-25/mconda3/setup.sh
conda activate envname
cd dirname
export PYTHONPATH='/full/path/to/dirname'
mpirun -x LD_LIBRARY_PATH -x PYTHONPATH -x PATH -np 20 -npernode 10 --hostfile $COBALT_NODEFILE python start_daskmpi.py cupy_test.py

        The number of processors per node (-npernode) should be (at least) the number of GPUs per node plus one.
        The number of processors (-np) should be -npernode times the number of nodes requested.

    Run a script in an interactive session

    1. ssh into one of Theta's login nodes

      ssh username@theta.alcf.anl.gov

    2. ssh into one of ThetaGPU's login nodes

      ssh thetagpusn1

    3. Submit an interactive job on n nodes

      qsub -n 2 -q full-node -A datascience -I -t 00:30:00

      A shell opens up on one of the compute nodes

    4. Load a pre-defined conda environment (either one works fine)

      with pytorch 

      source /lus/theta-fs0/software/thetagpu/conda/pt_master/2020-11-25/mconda3/setup.sh

      with tensorflow 

      source /lus/theta-fs0/software/thetagpu/conda/tf_master/2020-11-11/mconda3/setup.sh

    5. Activate the environment created in the install section

      conda activate envname

    6. cd into the directory dirname where files start_daskmpi.py and cupy_test.py are 

      cd dirname

    7. export the directory where cupy_test.py is to $PYTHONPATH

      export PYTHONPATH='/full/path/to/dirname'

    8. Run the example script cupy_test.py (taken from "GPU Dask Arrays, first steps: throwing Dask and CuPy together")

      • single node

        mpirun -np 9 python start_daskmpi.py cupy_test.py

        The number of processors (-np) should be (at least) the number of GPUs per node plus one.

        The output should be similar to the following: 

        Starting the scheduler

Starting the workers
Client status: <Client: 'tcp://172.23.2.199:8786' processes=8 threads=8, memory=270.50 GB>

    Start a JupyterLab interactive session

    1. ssh into one of Theta's login nodes

      ssh username@theta.alcf.anl.gov

    2. ssh into one of ThetaGPU's login nodes

      ssh thetagpusn1

    3. Submit an interactive job on n nodes

      qsub -n 2 -q full-node -A datascience -I -t 00:30:00

      A shell opens up on one of the compute nodes

    4. Load a pre-defined conda environment (either one works fine)

      with pytorch 

      source /lus/theta-fs0/software/thetagpu/conda/pt_master/2020-11-25/mconda3/setup.sh

      with tensorflow 

      source /lus/theta-fs0/software/thetagpu/conda/tf_master/2020-11-11/mconda3/setup.sh

    5. Activate the environment created in the install section

      conda activate envname

    6. cd into the directory dirname where file start_daskmpi.py is

      cd dirname

    7. Run the start_daskmpi.py script without any argument to start a JupyterLab session

      • single node

        mpirun -np 9 python start_daskmpi.py

        The number of processors (-np) should be (at least) the number of GPUs per node plus one.

      • multiple nodes

        mpirun -x LD_LIBRARY_PATH -x PYTHONPATH -x PATH -np 20 -npernode 10 --hostfile $COBALT_NODEFILE python start_daskmpi.py

        The number of processors per node (-npernode) should be (at least) the number of GPUs per node plus one.
        The number of processors (-np) should be -npernode times the number of nodes requested.

      The output should be similar to the following:

Starting the workers
Client status: <Client: 'tcp://172.23.2.203:8786' processes=8 threads=8, memory=270.50 GB>

    http://localhost:7787/?token=444f71ede62dfe1718f2499baee3483893fc14d6c7e97b88

    8. JupyterLab and Dask dashboard: You can connect to JupyterLab on http://localhost:7787/ in you browser and view the Dask dashboard on http://localhost:8787/status after you run the ssh command printed in the above output message in a shell on your local machine.
